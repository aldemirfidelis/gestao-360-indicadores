Como usar

Abra este arquivo no navegador para ver os fluxogramas renderizados. Use os botoes de cada diagrama para baixar SVG, ou use Imprimir / Salvar PDF para gerar uma versao de distribuicao.

Frontend Next.js 14, App Router, Tailwind, Recharts, React Flow, jsPDF e Papaparse.
Backend NestJS, Prisma, JWT, Zod, Helmet, Throttler e modulos de negocio.
Banco PostgreSQL com 41 entidades e multiempresa por companyId.
Ciclo principal Planejar, medir, analisar desvios, executar acoes e reportar.

1. Arquitetura

  W --> A["Cliente API

  B --> G["Guards globais

  M --> S["Package compartilhado
packages/shared"]
  S --> C["Enums, schemas Zod
calcStatus"]
  B -. pronto para filas .-> R[("Redis / BullMQ")]

Recharts, React Flow, jsPDF"]
  D["Deploy
Docker + DigitalOcean App Platform"] --> B
  D --> W
  D --> N[("Neon Postgres")]
  1. O usuario trabalha no frontend, que chama a API com JWT.
  2. O backend aplica autenticacao, perfis, validacao, CORS, helmet e throttling.
  3. Prisma centraliza acesso ao PostgreSQL; @g360/shared centraliza enums, Zod e farol.
  4. Redis/BullMQ esta preparado para evoluir notificacoes para fila ou cron.

DirecaoRegra de leitura
HIGHER_BETTERQuanto maior melhor; abaixo da meta usa tolerancia amarela antes de vermelho.
LOWER_BETTERQuanto menor melhor; acima da meta usa tolerancia amarela antes de vermelho.
EQUAL_TARGETQuanto mais perto da meta melhor.
RANGEVerde dentro da faixa; amarelo ou vermelho fora dela conforme tolerancia.

O desvio so e encerrado quando todas as acoes vinculadas estao concluidas. Isso evita que um problema seja marcado como resolvido sem execucao comprovada.
