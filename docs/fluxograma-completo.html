Como usar

Abra este arquivo no navegador para ver os fluxogramas renderizados. Use os botões de cada diagrama para baixar SVG, ou use Imprimir / Salvar PDF para gerar uma versão de distribuicao.

Frontend Next.js 14, App Router, Tailwind, Recharts, React Flow, jsPDF e Papaparse.
Backend NestJS, Prisma, JWT, Zod, Helmet, Throttler e módulos de negócio.
Banco PostgreSQL com 41 entidades e multiempresa por companyId.
Ciclo principal Planejar, medir, analisar desvios, executar ações e reportar.

1. Arquitetura

flowchart LR
  U["Usuário / Gestor"] --> W["Frontend Next.js 14
apps/web"]
  W --> A["Cliente API

  A -->|Bearer JWT| B["Backend NestJS

  B --> G["Guards globais
JWT + Roles + Throttler"]

  M --> S["Package compartilhado
packages/shared"]
  S --> C["Enums, schemas Zod
calcStatus"]
  B -. pronto para filas .-> R[("Redis / BullMQ")]
  W --> UI["UI
Tailwind, shadcn-style,
Recharts, React Flow, jsPDF"]
  D["Deploy
Docker + DigitalOcean App Platform"] --> B
  D --> W
  D --> N[("Neon Postgres")]
  1. O usuário trabalha no frontend, que chama a API com JWT.
  2. O backend aplica autenticacao, perfis, validação, CORS, helmet e throttling.
  3. Prisma centraliza acesso ao PostgreSQL; @g360/shared centraliza enums, Zod e farol.
  4. Redis/BullMQ esta preparado para evoluir notificações para fila ou cron.

HIGHER_BETTERQuanto maior melhor; abaixo da meta usa tolerância amarela antes de vermelho.
LOWER_BETTERQuanto menor melhor; acima da meta usa tolerância amarela antes de vermelho.
EQUAL_TARGETQuanto mais perto da meta melhor.
RANGEVerde dentro da faixa; amarelo ou vermelho fora dela conforme tolerância.

O desvio so e encerrado quando todas as ações vinculadas estão concluidas. Isso evita que um problema seja marcado como resolvido sem execucao comprovada.
